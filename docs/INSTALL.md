# Installation

### Prerequisites

* **Python 3.11+**
* **4GB RAM minimum** (8GB recommended)
* **10GB disk space** (for models)

### Step 1: Clone the repository

```bash
git clone https://github.com/RodriAF/RAG_ForStudyAndResearch.git
cd pdf-rag-app

```

### Step 2: Install Ollama

Ollama is required to run AI models locally.

#### On Linux:

```bash
curl -fsSL https://ollama.com/install.sh | sh

```

#### On macOS:

```bash
# Option 1: Download from
open https://ollama.com/download/mac

# Option 2: Using Homebrew
brew install ollama

```

#### On Windows:

```bash
# Download and install from
start https://ollama.com/download/windows

```

**Verify installation:**

```bash
ollama --version

```

### Step 3: Download an AI model

Choose one based on your hardware:

```bash
# Recommended for getting started (2GB - Fast)
ollama pull llama3.2

# More powerful but slower (4.7GB)
ollama pull llama3.1

# Optimized alternative for RAG (4.1GB)
ollama pull mistral

# Very lightweight and efficient (2.3GB)
ollama pull phi3

```

**Verify download:**

```bash
ollama list

```

### Step 4: Create project structure

```bash
# Create required __init__.py files
touch backend/app/__init__.py
touch backend/app/services/__init__.py

# On Windows, use these commands:
# type nul > backend\app\__init__.py
# type nul > backend\app\services\__init__.py

# Create data directories
mkdir -p data/uploads data/chroma_db

# On Windows:
# mkdir data\uploads
# mkdir data\chroma_db

```

### Step 5: Configure virtual environment

```bash
cd backend

# Create virtual environment
python -m venv venv

# Activate virtual environment
# On Linux/Mac:
source venv/bin/activate

# On Windows:
venv\Scripts\activate

```

### Step 6: Install Python dependencies

```bash
# Ensure you are in backend/ with venv activated
pip install --upgrade pip
pip install -r requirements.txt

```

**Note**: The first time you run it, it will download the embeddings model (~90MB). This is normal.

### Step 7: Configure environment variables

Create the `backend/.env` file:

```bash
# Create .env file
touch backend/.env  # On Windows: type nul > backend\.env

```

Edit `backend/.env` and add:

```env
# Ollama Configuration (Local)
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=llama3.2

# Embeddings Model (Local)
EMBEDDING_MODEL=sentence-transformers/all-MiniLM-L6-v2

# PDF Processing
CHUNK_SIZE=1000
CHUNK_OVERLAP=200

# Storage Paths
CHROMA_DB_PATH=./data/chroma_db
UPLOAD_DIR=./data/uploads

```

## Execution

### Option A: Manual Execution (3 terminals)

#### Terminal 1: Start Ollama

```bash
ollama serve

```

**Note**: On Mac/Windows, Ollama usually runs automatically as a background service. If it's already running, this step is not necessary.

#### Terminal 2: Start Backend

```bash
cd backend
source venv/bin/activate  # Windows: venv\Scripts\activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

```

You should see:

```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.

```

#### Terminal 3: Start Frontend

```bash
cd frontend
python -m http.server 8080

```

Then open your browser at: **http://localhost:8080**

### Option B: Open Frontend directly

If you prefer not to use an HTTP server:

1. Go to the `frontend/` folder
2. Double-click `index.html`
3. It will open in your default browser

### 3. Test the application

1. **Open the frontend** at `http://localhost:8080` or by opening `index.html`
2. **Upload a PDF** using the ""Choose File"" button
3. Click on **"Process PDF"**
4. **Wait** for it to process (you will see "✅ PDF processed: X chunks")
5. **Ask a question** in the text area
6. **Get a response** generated by local AI

## Usage

### Upload a document

1. Click on **"Choose File"**
2. Select your PDF file
3. Click on **"Process PDF"**
4. Wait for processing (may take 10-30 seconds depending on size)

### Ask questions

1. Select a document from the list
2. Type your question in the text area
3. (Optional) Adjust the number of chunks to use (1-10)
4. Click on **"Get Answer"**
5. Wait for the response (may take 5-20 seconds depending on your hardware)

### View relevant chunks

After receiving an answer, you can expand the "View relevant chunks" section to see which parts of the document the AI used to generate the response.

## Troubleshooting

### Error: "Ollama is not running"

**Solution:**

```bash
# Check status
ollama list

# Start Ollama
ollama serve

```

### Error: "Model not found: llama3.2"

**Solution:**

```bash
ollama pull llama3.2

```

### Error: "No module named 'app'"

**Solution:**

```bash
# Ensure you have the __init__.py files
touch backend/app/__init__.py
touch backend/app/services/__init__.py

```

### Error: "CORS policy blocked"

**Solution:**

* Verify that the backend is running
* Ensure CORS configuration exists in `backend/app/main.py`
* Try accessing the frontend via `http://localhost:8080` instead of `file://`

### Processing is very slow

**Solutions:**

* Use a lighter model: `ollama pull llama3.2` or `phi3`
* Reduce the number of chunks in queries (use 3 instead of 5)
* Close other applications to free up RAM
* Reduce `CHUNK_SIZE` in `.env` to `500`

### Out of memory / Insufficient RAM

**Solutions:**

* Use the lightest model: `phi3` (2.3GB)
* Close other applications
* Reduce `n_results` to 3 in queries
* If you have less than 4GB RAM, consider using Groq API (free) instead of local execution

## ⚙️ Advanced Configuration

### Change the AI model

1. **Download new model:**

```bash
ollama pull mistral  # or llama3.1, phi3, etc.

```

2. **Update `.env`:**

```env
OLLAMA_MODEL=mistral

```

3. **Restart backend**

### Adjust chunk size

In `.env`, modify:

```env
CHUNK_SIZE=500        # Smaller = more precision, slower
CHUNK_OVERLAP=100     # Overlap between chunks

```

### Change embeddings model

In `.env`:

```env
# Smaller (faster)
EMBEDDING_MODEL=sentence-transformers/all-MiniLM-L6-v2

# Better quality (slower)
EMBEDDING_MODEL=sentence-transformers/all-mpnet-base-v2

```
